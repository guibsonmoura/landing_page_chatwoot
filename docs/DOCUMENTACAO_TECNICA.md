## üè¢ **Sistema Multi-Tenant**

### **Arquitetura Multi-Tenant:**

O sistema implementa **multi-tenancy por linha** (row-level), onde:

1. **Cada usu√°rio** possui um `tenant_id`
2. **Todas as tabelas** referenciam `tenant_id`
3. **RLS** filtra automaticamente os dados
4. **Server Actions** sempre incluem verifica√ß√£o de tenant

### **Exemplo de Server Action Multi-Tenant:**
```typescript
export async function getAgents() {
  const supabase = createClient();
  const { data: { user } } = await supabase.auth.getUser();
  
  if (!user) throw new Error('Unauthorized');
  
  // Busca tenant do usu√°rio
  const { data: tenant } = await supabase
    .from('tenants')
    .select('id')
    .eq('user_id', user.id)
    .single();
    
  if (!tenant) throw new Error('Tenant not found');
  
  // Busca agentes do tenant (RLS aplica filtro autom√°tico)
  const { data: agents } = await supabase
    .from('tenant_agents')
    .select('*')
    .eq('tenant_id', tenant.id);
    
  return { data: agents };
}
```
